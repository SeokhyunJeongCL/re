현황 (AS-IS)
  • 키워드 태그 칩 불규칙 래핑 배치 / "칭찬해요" 혼재

#항목현황개선 방향우선순위
1 태그 칩 레이아웃 칩 길이 따라 불규칙 래핑 배치 2열 고정 그리드로 정렬하여 가독성·일관성 향상 P2
2 카테고리 전체 보기 위치 태그 칩 스크롤 후 하단에 위치 검색창 바로 아래 고정 노출하여 접근성 향상 P1
3 검색 결과 없음 화면 일관성 기타 문의·FAQ만 노출 (콜센터 연결 누락) 기타 문의 / FAQ / 콜센터 연결 3개 옵션 모두 일관 노출 P1
4 "칭찬해요" 태그 혼재 불편신고 플로우 내 칭찬 항목 노출 칭찬/건의 카테고리는 별도 진입점 분리 또는 시각적 구분 P2
5 진행 표시바 비율 Step 2 진행바 비율 불명확 3단계 기준 정확히 2/3(66%) 지점으로 표시 P3

UFMDS01840U — 상세 정보 입력 개선안
현황 (AS-IS)
  • 이미지 첨부 1장 제한 / 상세 요청 타입 카테고리 무관 고정 노출
  • 상세 장소 placeholder에 예시 없음 / 글자수 제한 미정의
  • 필수값 미입력 시 확인 버튼 자체가 미노출됨
#항목현황개선 방향우선순위
1 이미지 첨부 제한 1장만 첨부 가능 최대 3장으로 확대 고려 (서버 부하 기준 최종 결정). 버튼 옆 "최대 N장" 레이블 미리 노출 P2
2 요청 타입 동적 분기 모든 카테고리에 신규/변경·고장/장애 동일 노출 선택 카테고리에 따라 요청 타입 옵션 동적 변경 (예: 통근버스 민원은 미노출) P1
3 상세 장소 Placeholder "정확한 위치를 입력해주세요" 예시 텍스트 추가: "예: 703호 입구, 3층 복도 끝" P3
4 확인 버튼 동작 필수값 미입력 시 버튼 자체 미노출 항상 노출 + 미입력 시 disabled 처리, 탭 시 미입력 필드 강조 표시 P1
5 예약 날짜 선택 UI 날짜·시간 텍스트만 표시 Native Date/Time Picker 사용 방식 정의. 선택 가능 범위(최소: 당일+1, 최대: 30일) 명세 P2
6 글자수 카운터 미정의 textarea 우측 하단에 "0 / 200자" 형식으로 입력 가능 글자수 표시 P2
