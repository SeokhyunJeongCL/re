## 전체 플로우 도식

### 사용자(FO) 접수 플로우

```
[홈] 불편신고 진입
  └─▶ UFMDS01810U : 장소 선택 (Step 1/3)
        ├─ 건물 검색 탭 → UFMDS01820U : 건물 선택 Bottom Sheet
        │     └─ 건물 선택 완료 → 01810U 복귀 (건물명 세팅)
        ├─ 층 선택 (드롭다운)
        └─ 확인 버튼
              └─▶ UFMDS01830U : 카테고리/키워드 선택 (Step 2/3)
                    ├─ 태그 칩 선택 or 키워드 검색
                    ├─ [검색 결과 있음] → 01830U-1 : 필터된 키워드 + 도움 섹션
                    ├─ [검색 결과 없음] → 01830U-2 : 없음 안내 + 도움 섹션
                    ├─ [카테고리 전체 보기] → 01830U-3 : Bottom Sheet (카테고리 전체)
                    └─ 키워드/카테고리 선택 완료
                          └─▶ UFMDS01840U : 상세 정보 입력 (Step 3/3)
                                ├─ 이미지 추가 탭 → UFMDS01850U : 이미지 첨부 Bottom Sheet
                                │     ├─ [이미 1장 첨부 상태] → UFMDS01860U : 1장 제한 안내
                                │     └─ 사진 촬영 / 갤러리 선택
                                ├─ 일정 여부: 즉시 신청 / 예약 (날짜·시간 선택)
                                └─ 확인 버튼
                                      └─▶ UFMDS01880U : 접수 확인 Bottom Sheet
                                            ├─ [이전] → 01840U 복귀
                                            └─ [접수할게요]
                                                  └─▶ UFMDS01890U : 접수 완료
                                                        ├─ [콜센터 연결] → 전화 앱
                                                        └─ [접수 내역 확인] → 히스토리
```

### 사용자 접수 후 상태 플로우

```
접수 완료
  └─▶ 히스토리 화면: 불편신고 1건 표시
        └─▶ 히스토리 상세: 분류/접수시간/접수번호/접수위치/상세내용/사진
              └─ [접수 취소] → "접수가 취소되었습니다" 안내 (별도 확인 없이 즉시 취소)
```

### 관리자(BO) 처리 플로우

```
[불편신고 관리 목록]
  └─ 상태 필터: 전체 / 접수 / 접수확인 / 처리완료 / 취소
  └─ 기간 탭: 오늘 / 이번주 / 이번달 / 직접 입력
  └─ 추가 필터: 예약건만 보기 / 접수 24시간 이내 / 접수 24시간 경과
  └─ 건물 필터 드롭다운 (전체/특정 건물)
        └─▶ [불편신고 상세 — 접수 단계]
              ├─ 접수 내용 확인 (분류/접수시간/접수번호/접수위치/상세내용/처리희망시간/접수사진)
              ├─ 신청자 정보 + [신청자 연결] 버튼
              └─ [접수 확인] 버튼 → "접수확인 처리하시겠어요?" 다이얼로그
                    └─▶ 상태: 처리예정 | Push 알림 자동 발송
                          └─▶ [불편신고 상세 — 처리 단계]
                                ├─ 작업 내역 입력
                                │     ├─ 작업자명 (필수, 텍스트 입력)
                                │     ├─ 투입자재 (선택, textarea)
                                │     ├─ 작업 전 사진 (선택)
                                │     └─ 작업 후 사진 (선택)
                                ├─ 피드백 메시지 선택/작성
                                │     ├─ 기본 템플릿 선택 (5가지)
                                │     └─ 직접 작성
                                └─ [처리 예정] or [처리 완료] 버튼
                                      └─▶ [처리 완료 후]
                                            ├─ 접수 확인 시간/확인자
                                            ├─ 처리 완료 시간 (소요 시간)
                                            ├─ 작업자 정보
                                            ├─ 피드백 내용 확인
                                            └─ [수정] 버튼 (완료 후 수정 가능)
```

### UFMDS01810U — 장소 선택

**Step 1/3** | 화면명: "접수하기"

#### 화면 구성 요소

| 영역 | 컴포넌트 | 내용 |
|------|----------|------|
| 헤더 | 뒤로가기 + 타이틀 | `← 접수하기` |
| 진행 표시바 | Progress Bar | 3단계 중 1단계 (약 1/3 진행) — 분홍색 |
| 안내 텍스트 | 소제목 + 대제목 | `파인센터 불편 신고를 접수합니다` / `도움이 필요하신 장소를 골라주세요` |
| 건물 선택 | Input + 돋보기 아이콘 | 초기: `건물 선택` placeholder / 선택 후: `D&O 강서사옥` |
| 층 선택 | 드롭다운 | 초기: `층 선택` placeholder / 선택 후: `7F` |
| 가까운 건물 추천 팝업 | Tooltip 형태 카드 | 건물 필드 하단에 나타남 (조건부) |
| 하단 CTA | 버튼 | `확인` (분홍) — 건물+층 미선택 시 비활성 또는 탭 시 안내 |

#### 상태 분기

| 상태 | 화면 | 설명 |
|------|------|------|
| 초기 진입 (기본값 없음) | UFMDS01810U-1 | 건물 선택 / 층 선택 placeholder만 노출 |
| 기본 건물 세팅 상태 | UFMDS01810U | 최근 사용 건물(D&O 강서사옥), 층(7F) 자동 세팅 |
| 현위치≠기본 건물 | UFMDS01810U-11 | 팝업: "기본 건물이 현위치와 멀어요 / 가장 가까운 건물을 보여드릴게요!" + 건물명/주소/거리 + `이 건물로 변경할게요` 버튼 |

#### 개선안

| # | 항목 | 현황 | 개선 방향 | 우선순위 |
|---|------|------|-----------|---------|
| 1 | 층 선택 활성화 시점 | 건물 미선택 시에도 층 드롭다운 노출 및 탭 가능 상태 | 건물 선택 완료 후에만 층 드롭다운 활성화 (Sequential UX) — 건물 미선택 시 층 필드 비활성(회색) 처리 | 중 |
| 2 | 기본값 자동 세팅 | 매번 건물·층을 처음부터 선택해야 함 (기본값 없는 초기 상태) | 최근 사용한 건물·층을 기본값으로 자동 세팅 — 다를 경우에만 수정하도록 변경 | 상 |
| 3 | 현위치 감지 자동 팝업 | 현위치 기반 추천은 팝업 형태로만 제공되며 노출 조건 불명확 | **[확정]** 화면 진입 시 자동으로 현위치 기준 가장 가까운 건물 팝업 노출 — 사용자가 별도 버튼 탭 없이 즉시 확인 가능 | **상** |
| 4 | 팝업 버튼 레이블 | "이 건물로 변경할게요" — 어느 건물인지 모호 | 건물명 명시: `D&O 강남사옥으로 변경` 형태로 구체화 | 하 |
| 5 | 확인 버튼 비활성 처리 | 건물/층 미선택 시 버튼 상태 불명확 | 항상 노출하되 필수값 미입력 시 disabled 처리 + 탭 시 미입력 항목 강조 | 중 |
| 6 | 진행 단계 레이블 | 진행 표시바에 숫자만 있고 단계명 없음 | 진행바 아래 단계명 표시: `장소 선택 → 카테고리 → 상세 입력` | 하 |

### UFMDS01820U — 건물 선택 Bottom Sheet

**진입**: 01810U 건물 선택 필드 탭 시 하단에서 슬라이드업

#### 화면 구성 요소

| 영역 | 컴포넌트 | 내용 |
|------|----------|------|
| 헤더 | 타이틀 + 닫기 | `건물 선택` / `×` 버튼 |
| 검색 | Input + 돋보기 | `건물명 또는 장소를 입력해주세요` |
| 현위치 링크 | 텍스트 링크 | `⊕ 현위치로 근처 건물 보기` |
| 검색 결과 | 라디오 리스트 | 건물명 + 주소 + 거리 (예: D&O 강서사옥 / 서울특별시 강서구 마곡동 766 / 23m) |
| 하단 CTA | 버튼 | `확인` (분홍) |

#### 상태 분기

| 상태 | 화면 | 설명 |
|------|------|------|
| 초기 오픈 | UFMDS01820U | 검색창 빈 상태 — 현위치 기준 근처 건물 리스트 자동 로드 여부 불명확 |
| 검색 후 | UFMDS01820U-1 | 현위치 기반 건물 리스트 표시 (D&O 강서사옥 23m, LGSP_E9동 433m) |
| 검색 결과 없음 | (미정의) | 빈 화면 또는 안내 없음 |

#### 개선안

| # | 항목 | 현황 | 개선 방향 | 우선순위 |
|---|------|------|-----------|---------|
| 1 | 초기 기본 리스트 | Bottom Sheet 오픈 시 빈 상태 (검색 전 리스트 없음) | 오픈 시 최근 방문 건물 목록 또는 현위치 기반 가까운 건물 자동 노출 | 상 |
| 2 | 검색 결과 없음 처리 | 미정의 (빈 화면 가능성) | **[확정]** 검색 결과 없을 때 기타문의로 자동 연결 — 별도 선택 없이 기타문의 플로우로 직접 이동 | **상** |
| 3 | 검색 트리거 방식 | 검색 버튼 탭 필요 여부 불명확 | 실시간 검색(debounce 300ms) 방식으로 정의 — 입력과 동시에 결과 갱신 | 중 |
| 4 | 건물 상세 정보 | 건물명 + 주소 + 거리만 표시 | 건물 이미지 썸네일 또는 건물 유형(사옥/공장/상업시설) 아이콘 추가 고려 | 하 |
| 5 | 최근 검색 기록 | 없음 | Bottom Sheet 재오픈 시 최근 선택한 건물 우선 노출 | 중 |

### UFMDS01830U — 카테고리/키워드 선택

**Step 2/3** | 화면명: "접수하기" → 내용: "무엇을 도와드릴까요?"

#### 화면 구성 요소

| 영역 | 컴포넌트 | 내용 |
|------|----------|------|
| 헤더 | 뒤로가기 + 타이틀 | `← 접수하기` |
| 진행 표시바 | Progress Bar | 3단계 중 2단계 (약 2/3 진행) |

### 사용자 Pain Point — 개선 아이디어

#### 아이디어 1: 스마트 기본값 (Quick Start)
> 사용자가 불편신고 진입 시 **직전 접수 정보를 기본값**으로 세팅하고, 달라진 부분만 수정하도록 구성.
> - 같은 건물·층에서 일하는 직원은 매번 선택할 필요 없음
> - "전과 같은 위치에서 접수하시나요?" 원탭 컨펌 UX

#### 아이디어 2: 진행 상황 트래킹 위젯
> 접수 완료 후 홈 화면 또는 히스토리에 **처리 상태 진행바** 노출.
> ```
> [접수] ——● [접수확인] ——○ [처리중] ——○ [완료]
> ```
> 각 단계마다 Push 알림 + 앱 내 상태 표시

#### 아이디어 3: 접수 완료 화면 강화
> - 접수번호 크게 표시 (복사 가능)
> - 예상 처리 시간 (카테고리별 SLA 기반)
> - 처리 완료 후 만족도 평가 유도

| A-11 | 통계/분석 | 반복 신고 패턴 파악 불가 | 동일 위치/설비의 반복 고장 → 근본 원인 해결 아닌 반복 처리 | 상 |
| A-12 | 특수 건 처리 | 안전위험 신고 에스컬레이션 없음 | 위험 신고가 일반 불편신고와 동일 처리 흐름 → 즉시 대응 체계 부재 | 상 |

### 운영자 Pain Point — 개선 아이디어

#### 아이디어 1: 카테고리별 담당자 자동 배정
> 카테고리 설정에 따라 담당자 자동 배정.
> - 설비 고장 → 설비팀 / 통근버스 → 교통팀 / 안전위험 → 안전팀
> - 담당자 수신 즉시 Push 알림 발송

#### 아이디어 2: 긴급 건 에스컬레이션
> 안전위험 신고 또는 24시간 미처리 건에 대해 자동 에스컬레이션.
> - 안전위험 신고: 접수 즉시 담당 팀장에게 Push + 별도 표시
> - 24시간 미처리: 상위 관리자에게 자동 알림

#### 아이디어 3: 반복 신고 패턴 알림
> 동일 위치·설비에서 일정 기간 내 N회 이상 신고 시 알림.
> - "D&O 강서사옥 7F 에어컨이 이번 달 3회 신고되었습니다. 점검이 필요합니다."
> - 예방 정비(Preventive Maintenance) 트리거

#### 아이디어 4: 처리 성과 대시보드
> 관리자 홈 화면에 간단한 성과 지표 노출.
> - 오늘 처리율 / 평균 처리 시간 / 카테고리별 빈도 / 24시간 미처리 건수

#### 아이디어 5: 만족도 평가 연계
> 처리완료 알림에 만족도 평가 딥링크 포함.
> - 1~5점 평가 또는 이모지(😊😐😞)
> - 낮은 만족도(1~2점) 선택 시 추가 의견 입력창 + 관리자 재확인 알림
